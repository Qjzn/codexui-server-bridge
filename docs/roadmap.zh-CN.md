# 路线图

本路线图用于说明项目接下来最值得投入的方向，不承诺具体日期。

## 当前目标

让普通用户用最少配置，在电脑或服务器上稳定运行 Codex Web UI，并能从手机、局域网或受控远程入口访问。

## 第一阶段：部署更简单

- Cloudflare Tunnel 一键模式
- Windows 一条命令安装继续稳定
- 安装后输出清晰的本机地址、局域网地址、重启命令和配置路径
- 增加“让 Codex 自动部署”的提示词入口
- 安装失败时输出更清楚的排查建议

## 第二阶段：自托管远程访问

- frp 服务端包和客户端示例
- Nginx / Caddy 反向代理示例
- Tailscale / ZeroTier 使用说明
- 远程访问安全清单

## 第三阶段：Web 端体验稳定

- 长会话滚动继续减卡顿
- 会话切换动画和加载状态继续收口
- 执行中状态、待处理请求和命令输出继续贴近官方客户端体验
- 图片、本地文件、Markdown 链接展示更稳定
- 手机端输入区、技能选择和队列体验继续优化

## 第四阶段：运维与可观测性

- 服务健康检查更明确
- 日志目录和最近错误入口更容易找到
- 导出诊断包，方便用户反馈问题
- 常见故障自动提示修复命令

## 暂不优先

- 多用户权限系统
- 云端账号体系
- 大型数据库依赖
- 复杂团队协作功能

这些方向会显著增加部署门槛，与当前“新手一看就会，一条命令能跑”的目标冲突。
